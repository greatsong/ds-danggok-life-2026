🎯 오늘 만드는 것

1. 정직한 시험 — 안 본 종으로 재기 (백테스트)

모델을 학습에 쓴 종으로 다시 시험하면 점수가 부풉니다. 배운 답을 그대로 외웠을 수 있기 때문입니다. 진짜 실력은 학습에 안 쓴 종으로 재야 합니다. 이걸 백테스트라 합니다.

💡 외운 점수 vs 진짜 점수
  • 학습에 쓴 종만 다시 물으면 정확도가 거의 100%로 나옵니다 · 답을 외운 것이라 부풀린 점수입니다
  • 학습에서 뺀 종(안 본 종)으로 재면 약 87% · 이 쪽이 진짜에 가깝습니다
  • 9차시의 교차검증(약 83.5%)도 매번 안 본 조각으로 시험한 정직한 점수였습니다

2. 백테스트 + 배포 앱 (①②③)

① 샘플 프롬프트
스트림릿 '온혈·냉혈 판별 앱' 하나(main.py)를 만들어줘.
스트림릿 클라우드에 올릴 거야. 조건:

- 데이터는 이 주소에서 불러와(동물 4,513종의 몸 정보):
  https://raw.githubusercontent.com/greatsong/modudata/main/data/life_animals.csv
- 체중과 수명이 빈 종은 빼고, 포유류·새면 온혈, 나머지는 냉혈으로
  정답을 만들어줘. 빈칸은 중앙값으로 채워.
- 종의 4분의 1을 '안 본 종'으로 떼어 두고 나머지로만 학습시켜줘.
- 학습에 쓴 종의 정확도와, 떼어 둔 안 본 종의 정확도를 나란히 보여줘.
  두 숫자가 왜 다른지 한 줄로 설명도 붙여줘.
- 필요한 라이브러리 목록(requirements.txt)도 같이 줘.
  버전 숫자 없이 이름만, 스트림릿·판다스·넘파이는 기본 설치라 빼고.
- 한국어 라벨, 초보자용 한국어 주석. main.py 전체 코드를 한 번에 줘.
  • AI가 아래 모습의 코드를 줌 (똑같지 않아도 OK)
② AI가 준 코드 (예시)
import streamlit as st
import pandas as pd
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score

st.title("🧬 온혈·냉혈 판별 — 안 본 종으로 검증")

url = "https://raw.githubusercontent.com/greatsong/modudata/main/data/life_animals.csv"
df = pd.read_csv(url).dropna(subset=["adult_weight_g", "max_longevity_yrs"])
df["is_warm"] = df["class"].isin(["Mammalia", "Aves"]).astype(int)

feats = ["adult_weight_g", "max_longevity_yrs",
         "female_maturity_days", "gestation_days", "litter_size"]
X = df[feats].fillna(df[feats].median())
y = df["is_warm"]

# 종의 4분의 1을 '안 본 종'으로 떼어 두기
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=42)

model = RandomForestClassifier(random_state=42)
model.fit(X_train, y_train)

# 학습에 쓴 종(외운 것)과 안 본 종의 정확도
train_acc = accuracy_score(y_train, model.predict(X_train))
test_acc = accuracy_score(y_test, model.predict(X_test))
st.metric("학습에 쓴 종 정확도(외운 점수)", f"{train_acc:.1%}")
st.metric("안 본 종 정확도(진짜 점수)", f"{test_acc:.1%}")
⚠️ 두 정확도는 다르다
  • 학습에 쓴 종은 거의 100%, 안 본 종은 그보다 낮습니다 · 이 차이가 외운 점수와 진짜 점수의 거리입니다
  • 남에게 보여 줄 점수는 안 본 종으로 잰 것이어야 정직합니다
③ 프롬프트 개선 (예시)
좋아! 배포용으로 정리해줘.
- 체중·수명·성숙일수·임신일수·한 배 새끼 수를 넣으면 온혈인지 냉혈인지
  알려주는 화면으로 바꿔줘.
- 예측 옆에 "이 예측은 안 본 종 기준 정확도 약 87%짜리예요" 같은
  '얼마나 믿을지' 문구를 함께 보여줘.
- 냉혈은 자주 놓칠 수 있으니 참고하라는 안내도 넣어줘.

3. 스트림릿에 배포하기

  • 7차시 수명 예측기와 같은 방식 · 두 번째라 금방 됩니다
  1. 파일 두 개 준비
    main.py와 requirements.txt를 내 GitHub 저장소에 올리기
  2. 배포
    share.streamlit.io 접속 → Create app → 저장소·브랜치·main.py 선택 → Deploy
  3. 주소 공유
    배포가 끝나면 생긴 주소(URL)를 복사 → 갤러리에 게시 · 짝의 앱을 눌러 좋아요와 한 줄 피드백("정말 그런가?" 질문 포함)
🖥️ 화면이 다르게 보이면
  • 스트림릿·GitHub 화면 구성은 조금씩 바뀔 수 있어요 — 버튼을 못 찾으면 AI에게 물어보기("스트림릿 클라우드에 배포하는 지금 방법 알려줘")
