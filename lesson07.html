배포 — 내 예측기를 인터넷에

🎯 오늘 만드는 것
  • 수명 예측기(배포 URL) — 내 앱이 인터넷 주소를 얻어 누구나 열어 볼 수 있게 된다

0. 오늘의 목표

  • 6차시까지 우리 예측기는 한계까지 아는 앱이 됐습니다.
  • 이제 이걸 인터넷에 올려 주소 하나로 남과 나눕니다.
  • 배포 = 내 컴퓨터 안에만 있던 앱을 세상 어디서든 열 수 있게 하는 일입니다.

1. 배포 전 준비 — 파일 두 개

  • 배포에 꼭 필요한 파일은 두 개뿐입니다.
파일역할
main.py앱 본체 — 지금까지 만든 수명 예측기 코드
requirements.txt필요한 라이브러리 목록(scikit-learn·plotly)
  • 이 둘을 깃허브(GitHub) 저장소에 올려 두면, 스트림릿이 거기서 읽어 앱을 띄웁니다.
  • 깃허브 계정·저장소 만들기는 선생님 안내를 따릅니다(계정 정책은 학교마다 다릅니다).
💡 배포용으로 코드 다듬기 (①)
  • 먼저 앱을 한 화면에 깔끔히 정리하도록 AI에게 주문합니다.
① 샘플 프롬프트
지금까지 만든 동물 수명 예측기를 인터넷에 올릴 수 있게 한 화면으로 정리해 줘.
- 제목, 몸무게 슬라이더, 예측 수명, R² 점수, 외삽 경고 문구가 한 화면에 보이게 해 줘.
- 실제 점과 학습한 직선 그래프도 로그 축에 함께 넣어 줘.
- requirements.txt도 같이 줘. 버전 숫자 없이 이름만, 스트림릿·판다스·넘파이는 기본 설치라 빼고.
  • AI가 정리해 준 코드는 앞 차시들에서 이미 본 모습이라 여기선 다시 싣지 않습니다. (5차시 ② 참고)
③ 프롬프트 개선 예시
좋아, 이어서 고쳐 줘.
- 예측 몸무게가 데이터 범위를 벗어나면 '조심: 관측 밖 예측' 문구가 크게 뜨게 해 줘.
- 화면 맨 아래에 '이 예측을 어디까지 믿을지' 한 줄 안내를 넣어 줘(로그 척도·포유류 기준 등).

2. 스트림릿 클라우드에 올리기 (클릭)

  1. 배포 사이트 접속
    share.streamlit.io에 들어가 깃허브 계정으로 로그인. 처음이면 권한 허용을 눌러 줍니다
  2. 새 앱 만들기
    오른쪽 위 Create app(예전 이름: New app)을 누릅니다
  3. 내 저장소 고르기
    파일을 올려 둔 저장소(Repository)·브랜치(Branch)·파일 경로(main.py)를 고릅니다. 원하면 App URL에 기억하기 쉬운 주소를 정합니다
  4. 배포
    Deploy를 누르면 몇 분 안에 앱이 뜹니다. 생긴 주소(streamlit.app 으로 끝남)가 곧 내 예측기 주소
⚠️ 화면 구성은 조금씩 바뀔 수 있어요
  • 버튼 이름·위치는 스트림릿이 종종 바꿉니다. 못 찾으면 AI에게 "스트림릿 클라우드 배포 방법 알려 줘"라고 물어봅시다.
  • 배포는 처음엔 자주 실패합니다. 오류 메시지를 그대로 복사해 AI에게 보여 주면 대개 해결됩니다.
  • 코드를 고쳐 깃허브에 다시 올리면 앱도 자동으로 갱신됩니다.

3. 갤러리에 올리고 서로 검증

  1. 주소 게시
    배포된 앱 주소를 선생님이 안내하는 갤러리 주소에 올립니다
  2. 짝의 앱 눌러 보기
    다른 친구 앱을 열어 슬라이더를 움직여 봅니다. 좋아요와 한 줄 피드백을 남깁니다
  3. "정말 그런가?" 묻기
    피드백엔 칭찬만이 아니라 확인 한 줄을 꼭 넣습니다. 예: "포유류만 보면 답이 더 또렷하지 않아?"
💡 같은 데이터, 다른 결론
  • 남의 앱을 눌러 보면, 같은 동물 데이터인데 척도나 집단을 다르게 잡아 다른 결론을 낸 친구가 보입니다.
  • 누가 맞고 틀린 게 아니라, 어떤 조건에서 어떤 답인지가 다를 뿐입니다(6차시 세 조건).
⚠️ 오늘 산출물은 첫 증빙
  • 배포 주소는 수행평가와 갤러리의 첫 증빙입니다. 주소를 꼭 갤러리에 남깁시다.
  • 배포가 안 끝난 사람은 짝과 함께 마무리하거나 선생님 예비본으로 보충합니다.

🎒 오늘의 체크 & 링크

  • 배포엔 main.py와 requirements.txt 두 파일이 필요하다
  • 깃허브에 파일을 올리고 share.streamlit.io에서 앱을 배포했다
  • 내 앱이 streamlit.app 주소를 얻었다
  • 주소를 갤러리에 올리고 짝의 앱에 좋아요·한 줄 피드백을 남겼다
  • 피드백에 '정말 그런가?' 확인 한 줄을 넣었다

📚 오늘의 용어

오늘 나온 낱말 5개 — 외울 필요 없습니다 · 잊으면 여기로
용어뜻
배포내 앱을 인터넷에 올려 누구나 주소로 열 수 있게 하는 일

requirements.txt앱에 필요한 라이브러리 이름을 적어 두는 목록 파일
배포 URL배포된 앱의 인터넷 주소(streamlit.app으로 끝남)
