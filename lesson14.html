14차시 · 군집 트랙 마무리

이름은 사람이 붙인다

🎯 오늘 만드는 것
  • 동물 무리 성격 찾기 앱 — 동물 이름을 검색하면 소속 묶음과 그 묶음의 프로파일(체중·수명), 내가 붙인 이름을 보여 주는 앱 (배포까지)

0. 컴퓨터는 묶기만 한다

  • 지난 두 시간, 컴퓨터는 포유류를 몸 프로파일이 비슷한 묶음으로 나눴다.
  • 그런데 컴퓨터가 준 건 묶음 0, 묶음 1, 묶음 2라는 번호뿐이다. 그 번호엔 뜻이 없다.
  • 이 묶음이 "코끼리 같은 대형 장수형"인지 "생쥐 같은 소형 단명형"인지 읽어내는 건 사람의 몫이다. 오늘의 마지막 한 걸음.
⚠️ 묶음 번호는 실행마다 바뀐다
  • k-means의 묶음 번호(0·1·2)는 돌릴 때마다 순서가 달라질 수 있다.
  • 그러니 번호를 외우지 말고, 묶음의 프로파일(중앙값 체중·수명)을 보고 이름을 붙인다.

1. 묶음의 프로파일 읽기

  • 각 묶음의 중앙값 체중과 수명을 재면 성격이 보인다. 세 묶음의 중앙값은 이렇게 나온다.
중앙값 프로파일어떤 무리일까이름 후보
체중 약 70kg · 수명 약 30년크고 오래 사는 동물 (코끼리·고래·사람)대형 장수형
체중 약 2.5kg · 수명 약 18년중간 크기 동물 (개·여우 무리)중형
체중 약 66g · 수명 약 6년작고 짧게 사는 동물 (생쥐·땃쥐 무리)소형 단명형
💡 이름의 근거를 말로
  • "이 묶음은 중앙값이 70kg·30년이니 대형 장수형"처럼, 프로파일 숫자를 근거로 붙인다.
  • 이름은 정답이 아니라 해석이다. 근거만 또렷하면 다르게 불러도 된다.

2. 검색 앱 만들고 배포하기 (①②③)

  • 만들 것 = 동물 이름을 넣으면 소속 묶음 + 묶음 프로파일 + 내가 붙인 이름이 뜨는 앱
  • requirements.txt에는 scikit-learn만 (기본 설치 제외 · 버전 없이).
① 샘플 프롬프트
포유류 군집 앱을 배포용으로 정리해 줘. 스트림릿 클라우드에 올릴 거야. 조건:

- 포유류만 골라서 체중, 수명, 성숙 시기, 임신 기간 네 가지로,
  앞서처럼 로그와 표준화를 거쳐 k-means 3묶음으로 나눠 줘.
- 묶음마다 중앙값 체중을 재서, 큰 순서대로 대형 장수형, 중형,
  소형 단명형이라고 자동으로 이름을 붙여 줘.
- 위에 동물 이름을 입력하는 검색창을 둬. 동물을 넣으면 그 동물이
  어느 무리인지 알려 주고, 그 무리의 중앙값 체중과 수명도 보여줘.
- 한국어 라벨, 따뜻한 톤, 초보자용 한국어 주석. main.py 전체 코드를 한 번에 줘.
- 필요한 라이브러리 목록(requirements.txt)도 버전 없이 같이 줘.
② AI가 준 코드 예시
import streamlit as st
import pandas as pd
import numpy as np
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans

st.title("🧬 동물 무리 성격 찾기")

@st.cache_data
def load():
    url = "https://raw.githubusercontent.com/greatsong/modudata/main/data/life_animals.csv"
    return pd.read_csv(url)

df = load()

# 포유류만, 몸 프로파일 네 가지로 (로그와 표준화)
m = df[df["class"] == "Mammalia"]
feats = ["adult_weight_g", "max_longevity_yrs", "female_maturity_days", "gestation_days"]
a = m.dropna(subset=feats).copy()
Xs = StandardScaler().fit_transform(np.log10(a[feats]))
a["묶음"] = KMeans(n_clusters=3, random_state=42, n_init=10).fit_predict(Xs)

# 중앙값 체중이 큰 순서대로 이름을 붙인다 (번호는 실행마다 바뀌므로)
순서 = a.groupby("묶음")["adult_weight_g"].median().sort_values(ascending=False).index
이름 = {순서[0]: "대형 장수형", 순서[1]: "중형", 순서[2]: "소형 단명형"}
a["무리"] = a["묶음"].map(이름)

# 동물 검색
q = st.text_input("동물 이름을 입력하세요(영문)", "Human")
hit = a[a["common_name"] == q]
if len(hit) > 0:
    cl = int(hit["묶음"].iloc[0])
    st.subheader(f"{q} → {이름[cl]}")
    무리 = a[a["묶음"] == cl]
    st.write("이 무리의 중앙값 — 체중",
             f"{무리['adult_weight_g'].median():.0f} g · 수명",
             f"{무리['max_longevity_yrs'].median():.1f} 년")
else:
    st.info("그 동물을 찾지 못했어요. 영문 이름을 다시 확인해 주세요.")
✔️ 실행하면 이렇게 나온다
  • Blue whale·Human·Chimpanzee를 검색하면 대형 장수형에 들어간다.
  • Domestic dog를 검색하면 중형, 작은 땃쥐 무리는 소형 단명형에 들어간다.
  • 중앙값을 보고 이름이 대형 장수형·중형·소형 단명형으로 자동으로 붙으면 완성.
③ 프롬프트 개선 예시
좋아! 배포용으로 마무리해 줘.
- 세 무리의 중앙값 체중과 수명을 한눈에 보는 작은 표를 위에 띄워 줘.
- 검색한 동물이 세 무리 중 어디에 드는지 눈에 띄게 강조해 줘.

  라는 한 줄 안내를 넣어줘.

2-1. 배포하기 (클릭)

  1. 깃허브에 올리기
    main.py와 requirements.txt를 내 깃허브 저장소에 올린다.
  2. 스트림릿 연결
    share.streamlit.io에서 New app → 저장소·main.py 선택 → Deploy. 준비에 1~2분. (화면 구성은 조금씩 바뀔 수 있어요 — 못 찾으면 AI에게 물어보기)
  3. 주소 확인·게시
    생긴 앱 주소(URL)를 갤러리에 올리고, 짝의 앱을 눌러 좋아요와 한 줄 피드백을 남긴다.
💡 사람이 이상치가 될 때
  • Human은 체중 62kg, 수명 122년이라 대형 장수형에 든다. 몸집에 비하면 유난히 오래 사는 종이다.
  • 6차시에서 본 "크기 대비 오래 사는 종"(박쥐·사람)이 여기서도 얼굴을 내민다. 데이터는 여러 트랙에서 같은 이야기를 들려준다.

3. 군집 트랙 마무리

  • 비지도학습으로 동물을 묶고(12차시), 그 묶음을 언제 믿을지 확인하고(13차시), 오늘 뜻을 붙였다.
  • 데이터 과학의 마지막 한 걸음은 계산이 아니라 해석이다.
✔️ 세 트랙을 지나며 배운 것
  • 회귀: 직선을 그어도 언제 믿을 수 있나를 먼저 묻는다.
  • 분류: 정확도가 높아도 무엇을 놓쳤나를 본다.
  • 군집: 점수가 정답을 주지 않기에 사람이 이름을 붙인다.
📝 수행 과제 — 이름의 근거
  • 내가 붙인 묶음 이름 세 개와, 각 이름을 붙인 근거(중앙값 체중·수명)를 한 줄씩 적는다.
  • 배포한 앱 주소를 갤러리에 올린다.
