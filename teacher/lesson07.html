7차시 · 교사용

배포 · 내 예측기를 인터넷에 — 교사 진행안

학생용 7차시 페이지 ↗를 화면에 띄운 채 진행합니다. 여기에는 시간표 · 멘트 · 예상 질문 · 예상 오류 · 이론 심화를 덧댔습니다.

🎙️ 개강 멘트 — 첫 배포, 첫 증빙

“6차시까지 우리 예측기는 한계까지 아는 앱이 됐어요. 오늘은 이걸 인터넷에 올려 주소 하나로 남과 나눕니다. 오늘 나온 배포 주소가 수행평가와 갤러리의 첫 증빙이에요.” 배포는 실패가 잦으니 짝 페어링과 예비 배포본을 미리 준비.

⏱️ 50분 진행 시간표

시간블록핵심
0~8′복습·도입파일 두 개(main.py·requirements.txt) · 배포 = 세상 어디서든 열기
8~30′①②③ 실습 (배포)배포용 정리 → 깃허브 업로드 → share.streamlit.io 배포 → 주소 생성
30~45′상호 검증갤러리에 URL 게시 · 짝의 앱 눌러 좋아요 + "정말 그런가?" 한 줄
45~50′정리 (이월 가능)배포 실패자 보충 안내
🎙️ 개강 전 확인 · 계정 정책

깃허브·스트림릿 학생 계정 생성 허용 여부와 연령 정책은 학교마다 다릅니다(설계문서 개강 전 확인 항목). 미확인이면 선생님 계정으로 대표 배포하거나 짝당 한 계정으로 운영. 갤러리 주소는 선생님이 안내(Supabase, 장애 시 패들렛 폴백).

블록별 진행 노트

①②③ 실습 — 배포 (8~30′)

📖 이론 심화 — 외삽 경고를 왜 앱에 넣나 (지도서 5장 참고)

배포 화면에 ‘조심: 관측 밖 예측’ 문구를 넣는 건 미관이 아니라 통계입니다. 예측 분산 Var(ŷ₀) = σ²[1/n + (x₀−x̄)²/S_xx]는 x₀(로그 몸무게)가 관측 평균에서 멀수록 제곱으로 커집니다. 데이터가 거의 없는 극단 몸무게(벌새보다 가볍거나 대왕고래보다 무거운 값)는 이 식이 경고하는 지대죠. 게다가 우리 모델은 멱함수라는 형태가 범위 밖에서도 유지된다는 가정에 전적으로 기댑니다. 실제 생물엔 상한이 있어 무한정 오래 사는 종은 없는데, 직선은 끝없이 올라가죠. 여기에 6차시가 밝힌 척도·집단 의존성과 예외(박쥐·사람)가 겹칩니다 — 같은 몸무게라도 어느 집단·어느 종이냐에 따라 실제 수명이 크게 벌어집니다. 그래서 예측 옆의 딱지는 “이 숫자는 포유류·로그 척도·관측 범위라는 조건 위의 가정”이라는 정직한 고지입니다. 지도서 5장 5.4가 근거.

🧯 예상 오류 (실측 기반 · 배포 빈출)

requirements.txt 누락/철자 → 설치명 scikit-learn·plotly. 스트림릿·판다스·넘파이는 기본 설치라 빼되, sklearn·plotly를 안 적으면 배포 서버에서 ModuleNotFound. · 파일 경로 불일치 → 배포 설정의 파일 경로가 실제 main.py와 같은지(대소문자·확장자). · 배포 첫 시도 실패 → 정상입니다. 오류 메시지를 그대로 복사해 AI에게 붙이면 대개 해결. 학생이 겁먹지 않게 “처음엔 원래 실패한다” 미리 안내. · 버튼 이름이 교재와 다름 → 스트림릿이 UI를 종종 바꿈(New app→Create app 등). 못 찾으면 AI에 “스트림릿 클라우드 배포 방법”을 물어보게. · 데이터 URL 접근 차단 → 배포 서버에서 raw.githubusercontent.com이 되는지(사내망 아님이라 대개 됨).

🙋 예상 질문

“돈 들어요?” → “스트림릿 커뮤니티 클라우드는 무료 한도 안에서 해요.” · “코드를 고치면 다시 배포해야 해요?” → “깃허브에 다시 올리면 앱이 자동으로 갱신돼요. 배포는 한 번, 이후엔 코드만 고치면 끝.” · “배포가 계속 실패해요” → “오류 메시지를 그대로 AI에게 보여 주세요. 대부분 requirements나 파일 경로 문제라 금방 잡혀요.”

상호 검증 (30~45′)

🎙️ 멘트 — 칭찬 말고 확인 한 줄

“짝의 앱을 눌러 보세요. 같은 동물 데이터인데 척도나 집단을 다르게 잡아 다른 결론을 낸 친구가 보일 거예요. 누가 맞고 틀린 게 아니라 어떤 조건에서 어떤 답인지가 다를 뿐이죠(6차시 세 조건). 피드백엔 칭찬만이 아니라 확인 한 줄을 꼭 넣으세요 — ‘포유류만 보면 답이 더 또렷하지 않아?’처럼.”

정리 (45~50′ · 이월 가능)

🎙️ 마무리 — 못 끝낸 사람

“배포가 안 끝난 사람은 짝과 함께 마무리하거나 선생님 예비본으로 보충해요. 주소는 꼭 갤러리에 남기고요. 이건 수행평가 첫 증빙이에요.”

🧾 생기부 관찰 지점

결과 해석 · 한계 인식 — 짝의 앱을 눌러 보고 “척도나 집단을 다르게 잡으면 결론이 달라진다”는 비판적 확인 한 줄을 근거와 함께 남긴 학생, 또는 배포 실패를 오류 메시지로 진단해 해결한 학생을 채집.

✅ 7차시 마무리 체크 (교사)

  • 전원(또는 짝당 1개) 예측기가 streamlit.app 주소를 얻었나? (실패자 → 짝·예비본)
  • 외삽 경고 문구가 배포 화면에 들어갔나?
  • 주소를 갤러리에 게시하고 짝의 앱에 확인 한 줄을 남겼나?
  • 배포 URL 과제를 클래스룸 + 갤러리에 수집했나?
