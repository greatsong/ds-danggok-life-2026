  • 온혈·냉혈 분류기 초안 — 동물의 몸 정보를 보고 온혈인지 냉혈인지 맞히는 첫 모델

0. 숫자를 맞히던 회귀에서, 갈래를 맞히는 분류로

지난 8차시까지 회귀는 "이 동물은 몇 년을 살까"처럼 숫자를 맞혔습니다. 오늘부터 다루는 분류는 답이 숫자가 아닙니다. "이 동물은 온혈일까 냉혈일까"처럼 정해진 몇 개의 갈래 중 하나를 맞힙니다.

  • 답이 숫자 하나로 나오면 회귀, 몇 개의 칸 중 하나로 나오면 분류
  • "동물의 수명 예측"은 회귀, "온혈/냉혈 판별"은 분류 · "멸종위기 여부"도 분류

1. 정답 만들기 — 온혈은 누구인가

이 데이터에는 온혈·냉혈이라고 적힌 열이 따로 없습니다. 그래서 분류(class) 열로 정답을 직접 만듭니다. 포유류와 새는 스스로 체온을 유지하는 온혈, 나머지(어류·파충류·양서류)는 냉혈로 묶습니다.

  • 파일 = life_animals.csv · 동물 4,513종의 몸 정보(체중·수명·성숙 기간·임신 기간 등)
  • 회귀 때처럼 체중·수명이 빈 종을 먼저 빼면 3,188종이 남습니다. 오늘은 이 3,188종으로 모델을 만듭니다.
  • 이 데이터는 modudata 저장소에 올라가 있습니다 → github.com/greatsong/modudata

2. 첫 분류 모델 돌려 보기 (①②③)

  • 도구 = scikit-learn의 랜덤 포레스트(여러 판단 나무의 다수결로 갈래를 고름)
  • 정확도는 데이터를 다섯 조각으로 나눠 번갈아 시험한 평균으로 잽니다(5겹 교차검증). 한 번만 나누는 것보다 들쭉날쭉이 적습니다.
  • requirements.txt엔 scikit-learn만 — 스트림릿·판다스·넘파이는 기본 설치, 버전 없이

① 샘플 프롬프트
스트림릿 '온혈·냉혈 분류기' 앱 하나(main.py)를 만들어줘. 조건:

- 데이터는 이 주소에서 불러와(동물 4,513종의 몸 정보야. 열은 이름,
  강, 목, 성체 체중, 최대 수명, 대사율, 체질량, 암컷 성숙일수,
  임신일수, 한 배 새끼 수, 체온):
  https://raw.githubusercontent.com/greatsong/modudata/main/data/life_animals.csv
- 체중과 수명이 빈 종은 먼저 빼줘(3,188종 남아).
- 강이 포유류나 새면 온혈, 나머지는 냉혈으로 정답을 만들어줘.
- 체중·수명·성숙일수·임신일수·한 배 새끼 수를 보고 온혈인지 냉혈인지
  맞히는 랜덤 포레스트 분류기를 만들어줘. 빈칸은 중앙값으로 채워.
- 데이터를 다섯 조각으로 나눠 번갈아 시험하는 방식(교차검증)으로
  정확도를 재서 큰 숫자 카드로 보여줘.
- 필요한 라이브러리 목록(requirements.txt)도 같이 줘.
  버전 숫자 없이 이름만, 스트림릿·판다스·넘파이는 기본 설치라 빼고.
- 한국어 라벨, 초보자용 한국어 주석. main.py 전체 코드를 한 번에 줘.
  • AI가 아래 모습의 코드를 줌 (똑같지 않아도 OK)
② AI가 준 코드 (예시)
import streamlit as st
import pandas as pd
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import cross_val_predict
from sklearn.metrics import accuracy_score

st.title("🧬 온혈? 냉혈? 첫 분류 모델")

# 데이터 불러오기 — 동물 4,513종의 몸 정보
url = "https://raw.githubusercontent.com/greatsong/modudata/main/data/life_animals.csv"
df = pd.read_csv(url)

# 체중·수명이 빈 종은 빼기(3,188종 남음) — 회귀 때 쓰던 그 손질
df = df.dropna(subset=["adult_weight_g", "max_longevity_yrs"])

# 정답 만들기: 포유류·새면 온혈(1), 나머지는 냉혈(0)
df["is_warm"] = df["class"].isin(["Mammalia", "Aves"]).astype(int)

# 입력(특징): 몸에 관한 다섯 가지 숫자
feats = ["adult_weight_g", "max_longevity_yrs",
         "female_maturity_days", "gestation_days", "litter_size"]
X = df[feats].fillna(df[feats].median())   # 남은 빈칸은 중앙값으로
y = df["is_warm"]

# 랜덤 포레스트 + 5겹 교차검증으로 정확도 재기
model = RandomForestClassifier(random_state=42)
pred = cross_val_predict(model, X, y, cv=5)
acc = accuracy_score(y, pred)
st.metric("정확도", f"{acc:.1%}")
💡 방금 무슨 일이?
  • 정답이 있는 데이터로 배웠습니다 = 지도학습 (회귀도 지도학습이었습니다)
  • 회귀의 model.fit()과 뿌리가 같은 도구 — 바뀐 건 "숫자" 대신 "갈래"를 맞힌다는 것뿐
  • 첫 정확도는 약 83.5% — 열에 여덟쯤은 맞힌 셈
  • 어느 갈래를 더 잘 맞히는지 보고 싶으면 대화로 개선 ↓

⚠️ 잠깐 — 83.5%면 정말 잘한 걸까?

  • 온혈(포유류·새)은 3,188종 중 2,398종 · 이 사실이 다음 시간 정확도만 보면 놓치는 것으로 이어집니다
