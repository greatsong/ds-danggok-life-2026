정답표 없이 비슷한 것끼리 묶기

🎯 오늘 만드는 것
  • 포유류 군집 초안 — 정답표 없이, 몸 프로파일이 닮은 포유류끼리 묶는 앱

0. 회귀·분류엔 정답이 있었다

회귀는 체중에서 수명이라는 정답을, 분류는 몸 정보에서 온혈·냉혈이라는 정답을 보고 규칙을 배웠습니다. 오늘은 정답표가 없습니다. 컴퓨터가 스스로 비슷한 것끼리 묶습니다. 이것이 비지도학습입니다.

종류정답표하는 일예
지도학습있음정답을 보고 규칙을 배움수명 회귀 · 온혈/냉혈 분류
비지도학습없음스스로 비슷한 것끼리 묶음오늘: 포유류 군집

1. 오늘의 데이터 — 포유류의 몸 프로파일

  • 포유류(Mammalia)만 골라, 체중·수명·성숙일수·임신일수 네 가지가 다 있는 종만 남기면 784종이 됩니다.
  • 몸집 큰 코끼리부터 작은 생쥐까지, 종마다 이 네 숫자의 조합이 다릅니다. 비슷한 조합끼리 묶어 봅니다.
  • 이 데이터는 modudata 저장소에 올라가 있습니다 → github.com/greatsong/modudata
💡 왜 로그로 눕히고 단위를 맞출까
  • 체중은 몇 그램부터 수십만 그램까지 여덟 자릿수를 오갑니다. 그대로 재면 체중 하나가 거리를 다 지배합니다.
  • 그래서 회귀 때처럼 로그로 눕히고, 네 숫자의 단위를 나란히 맞춥니다(표준화). 이래야 네 가지가 공평하게 묶기에 쓰입니다.
  • 이 손질을 건너뛰면 어떻게 되는지는 다음 시간에 직접 봅니다.

2. 첫 군집 만들기 (①②③)

  • 도구 = k-means(케이민즈): 미리 정한 묶음 수 k개로 비슷한 것끼리 나누는 방법.
  • 잘 나뉜 정도는 실루엣 점수로 잽니다. 1에 가까울수록 안이 촘촘하고 묶음끼리 멉니다.
  • requirements.txt에는 scikit-learn만 (스트림릿·판다스·넘파이는 기본 설치 · 버전 없이).
① 샘플 프롬프트
스트림릿 '포유류 군집' 앱 하나(main.py)를 만들어줘. 스트림릿 클라우드에
올릴 거야. 조건:

- 데이터는 이 주소에서 불러와(동물 4,513종의 몸 정보):
  https://raw.githubusercontent.com/greatsong/modudata/main/data/life_animals.csv
- 포유류만 남기고, 체중·수명·성숙일수·임신일수 네 가지가 다 있는 종만
  골라 줘(784종쯤 돼).
- 네 숫자는 크기 차이가 크니 로그로 눕히고 단위를 맞춰 줘(표준화).
  크기 하나가 묶기를 지배하지 않게.
- k-means로 비슷한 몸 프로파일의 포유류끼리 3묶음으로 나눠 줘.
- 얼마나 잘 나뉘었는지 실루엣 점수를 카드로 크게 보여주고,
  이 점수가 뭔지 한 줄로 설명해 줘.
- 묶음마다 종이 몇 개씩 들어갔는지도 보여 줘.
- 필요한 라이브러리 목록(requirements.txt)도 같이 줘.
  버전 숫자 없이 이름만, 스트림릿·판다스·넘파이는 기본 설치라 빼고.
- 한국어 라벨, 따뜻한 톤, 초보자용 한국어 주석. main.py 전체 코드를
  한 번에 줘.
  • AI가 아래 모습의 코드를 준다 (똑같지 않아도 된다).
② AI가 준 코드 예시
import streamlit as st
import pandas as pd
import numpy as np
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score

st.title("🧬 포유류를 몸 프로파일로 묶기")

url = "https://raw.githubusercontent.com/greatsong/modudata/main/data/life_animals.csv"
df = pd.read_csv(url)

# 포유류만, 네 가지 몸 정보가 다 있는 종만(784종쯤)
feats = ["adult_weight_g", "max_longevity_yrs",
         "female_maturity_days", "gestation_days"]
m = df[df["class"] == "Mammalia"].dropna(subset=feats)

# 크기 차이가 크니 로그로 눕히고, 단위를 맞춰(표준화) 거리 왜곡을 줄이기
X = np.log10(m[feats])
Xs = StandardScaler().fit_transform(X)

# k-means로 3묶음
labels = KMeans(n_clusters=3, random_state=42, n_init=10).fit_predict(Xs)

# 얼마나 잘 나뉘었나 — 실루엣 점수(1에 가까울수록 좋음)
score = silhouette_score(Xs, labels)
st.metric("실루엣 점수 (1에 가까울수록 잘 나뉨)", f"{score:.2f}")

# 묶음별 종 수
st.subheader("묶음별 종 수")
st.write(pd.Series(labels, index=m.index).value_counts().sort_index())
✔️ 실행하면 이렇게 나온다
  • 포유류 784종, 실루엣 점수 0.35 안팎
  • 묶음 크기는 대략 274 · 307 · 203 세 무리로 갈립니다
  • 이 데이터는 고정된 기록이라 random_state를 같게 두면 숫자도 같게 나옵니다
③ 프롬프트 개선 예시
좋아! 이어서 고쳐줘.
- 묶음마다 대표 종 몇 개(그 묶음에서 눈에 띄는 종)를 함께 보여줘.
- 묶음마다 체중과 수명의 중앙값을 표로 보여줘. 어떤 무리가 크고 오래
  사는 무리인지 한눈에 알게.

3. 눈에 걸리는 것 하나

정답표가 없는데도 컴퓨터가 포유류를 세 무리로 묶었습니다. 신기한 순간입니다. 그런데 두 가지 질문이 남습니다.

  • 왜 하필 3묶음일까? 4묶음이면 달라질까?
  • 실루엣 점수만 보면 2묶음이 더 높다던데, 그럼 3묶음이 틀린 걸까?
⚠️ 오늘은 답하지 않는다 (다음 시간 재료)
  • 이 두 질문, 즉 "묶음 수 k의 정답"과 "실루엣 점수와 해석이 어긋날 때"는 13차시에서 자세히 다룹니다.
  • 지금은 "군집엔 하나의 정답이 없다"는 낌새만 챙깁니다.
