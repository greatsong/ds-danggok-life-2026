점수는 정답을 주지 않는다

“모든 모델은 틀렸다. 다만 몇몇은 쓸모가 있다.”
조지 박스 · “All models are wrong, but some are useful.” (1976)
🎯 오늘 하는 것
  • k 비교표 — 2·3·4묶음을 나란히 두고 실루엣 점수·묶음 크기·이상치를 견주어, 어느 묶음을 언제 믿을지 판단으로 적는 기록

0. 지난 시간에 남긴 두 질문

  • 포유류를 몸 프로파일(체중·수명·성숙 시기·임신 기간)로 3묶음으로 나눴고, 실루엣은 0.35 안팎이었다.
  • 남은 질문 둘: ① 왜 하필 3묶음일까? ② 왜 로그와 표준화를 거쳤을까?
  • 오늘은 이 두 질문을 직접 확인한다. 세 가지 짚을 점이 모두 "사람의 판단이 필요하다"로 모인다.

1. 짚을 점 하나 — k에는 정답이 없다

  • 묶음 수 k를 바꾸면 점수도 결과도 달라진다. 2·3·4묶음의 실루엣 점수를 나란히 견주어 보자.
① 샘플 프롬프트
지난번 포유류 군집을 이어서 살펴봐 줘.
- 같은 데이터를 2묶음, 3묶음, 4묶음으로 각각 나눠서
  실루엣 점수와 묶음별 동물 수를 나란히 비교해 줘.
- 점수가 가장 높은 묶음 수가 몇인지도 알려 줘.
② AI가 준 코드 예시
import pandas as pd
import numpy as np
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score

URL = "https://raw.githubusercontent.com/greatsong/modudata/main/data/life_animals.csv"
df = pd.read_csv(URL)

# 포유류만, 몸 프로파일 네 가지로 (12차시와 같은 준비)
m = df[df["class"] == "Mammalia"]
feats = ["adult_weight_g", "max_longevity_yrs", "female_maturity_days", "gestation_days"]
a = m.dropna(subset=feats)

X = np.log10(a[feats])            # 로그로 크기 차이를 눌러 주고
Xs = StandardScaler().fit_transform(X)   # 표준화로 단위를 맞춘다

for k in [2, 3, 4]:
    labels = KMeans(n_clusters=k, random_state=42, n_init=10).fit_predict(Xs)
    score = silhouette_score(Xs, labels)
    sizes = pd.Series(labels).value_counts().sort_index().tolist()
    print(f"k={k} · 실루엣 {score:.3f} · 묶음 크기 {sizes}")
✔️ 실행하면 이렇게 나온다
  • k=2 · 실루엣 0.44 · 묶음 크기 [452, 332]
  • k=3 · 실루엣 0.35 · 묶음 크기 [274, 307, 203]
  • k=4 · 실루엣 0.29 · 묶음 크기 [112, 231, 262, 179]
  • 점수는 2묶음이 가장 높다. 그런데 우리는 3묶음을 골랐다 (이유는 바로 아래).
⚠️ 점수가 높은 답 ≠ 사람이 이해하는 답
  • 실루엣만 보면 "2묶음이 정답"이다. 하지만 2묶음은 세상을 큰 동물·작은 동물 딱 둘로만 뭉뚱그린다.
  • 3묶음으로 늘리면 대형 장수형·중형·소형 단명형으로 갈라져 성격이 또렷해진다 (다음 시간에 이름을 붙인다).
  • k에 절대 정답은 없다. 점수와 해석을 함께 보고 사람이 고른다.

2. 짚을 점 둘 — 이상치 하나가 저울을 기울인다

  • 이 데이터에서 가장 무거운 동물은 흰긴수염고래(Blue whale)다. 체중이 무려 136톤, 즉 136,000,000g이다.
  • 가장 가벼운 포유류는 몇 g밖에 안 된다. 체중이 여덟 자릿수나 차이 난다.
  • 이렇게 혼자 멀리 떨어진 값을 이상치라 한다. 아무 손질 없이 묶으면, 이 거인 하나가 묶기 전체를 끌고 간다.
💡 이상치를 어떻게 볼까
  • 흰긴수염고래는 잘못 적힌 값이 아니라 진짜로 그만큼 큰 동물이다. 지워야 할 오류가 아니다.
  • 문제는 값 자체가 아니라 재는 방식이다. 원래 체중 그대로 재면 고래 하나가 저울을 다 차지한다.
  • 그래서 필요한 것이 다음 짚을 점, 정규화다.

3. 짚을 점 셋 — 무엇으로 재느냐가 결과를 바꾼다 (★)

  • 12차시에서 우리는 체중을 로그로 바꾸고 표준화했다. 그 손질을 빼면 어떻게 될까? 직접 비교해 보자.
③ 프롬프트 개선 예시
이번엔 로그와 표준화를 빼고 비교해 줘.
- 체중·수명 같은 원래 숫자 그대로 3묶음으로 나눠 봐.
- 각 묶음에 동물이 몇 마리 들어갔는지, 가장 작은 묶음엔
  어떤 동물이 있는지 이름을 보여 줘.
- 손질했을 때와 안 했을 때 묶음이 어떻게 달라지는지 나란히 봐.
② AI가 준 코드 예시
# 로그·표준화 없이 — 원래 숫자 그대로 3묶음으로
labels_raw = KMeans(n_clusters=3, random_state=42, n_init=10).fit_predict(a[feats])
groups = pd.Series(labels_raw, index=a["common_name"].values)

sizes = groups.value_counts().sort_index()
print("묶음 크기:", sizes.tolist())

small = sizes.idxmin()          # 가장 작은 묶음
print("가장 작은 묶음의 동물:", list(groups[groups == small].index))
✔️ 실행하면 이렇게 나온다
  • 손질 없이 묶으니 묶음 크기가 [775, 2, 7]로 나온다. 한 묶음에 775마리가 몰린다.
  • 가장 작은 묶음엔 흰긴수염고래·북극고래뿐이다. 거대한 고래들끼리만 따로 떨어져 나왔다.
  • 체중 숫자가 워낙 커서, 컴퓨터 눈엔 고래냐 아니냐만 보인다. 나머지 미세한 차이는 다 묻혔다.
⚠️ 무엇을 기준으로 재느냐가 결과를 정한다
  • 같은 데이터, 같은 k인데 재는 방법만 바꿔도 묶음이 완전히 달라진다.
  • 로그와 표준화는 덩치의 격차를 눌러 모든 특징을 공평하게 견주게 해 준다. 그래서 12차시에 그 손질을 먼저 했다.
  • "덩치로 묶을까, 삶의 방식으로 묶을까"는 사람이 던지는 질문이다. 컴퓨터는 시킨 대로 잴 뿐이다.

4. 오늘의 정리 — 세 번의 확인

짚을 점무엇을 확인했나사람이 할 판단
k의 자의성점수는 2묶음, 해석은 3묶음점수와 뜻을 함께 보고 고르기
이상치흰긴수염고래 136톤이 저울을 기울임왜 튀는지 알고 다룰 방법 정하기
정규화로그·표준화를 빼면 덩치로만 묶임무엇으로 잴지 질문 세우기
🧬 점수가 정답을 주지 않아도 괜찮다
  • "정답이 없다"는 건 아무렇게나 해도 된다는 뜻이 아니다.
  • 점수가 정답을 대신 주지 않기에 사람의 판단이 필요하다. 다음 시간, 그 판단으로 묶음에 이름을 붙인다.
📝 수행 과제 — k 비교표
  • 2·3·4묶음의 실루엣 점수와 묶음 크기를 표로 적는다.
  • 어느 k를 고를지와 그 이유(점수 근거 + 해석 근거)를 두세 줄로 쓴다.
  • 흰긴수염고래가 왜 이상치인지, 정규화가 왜 필요한지 각각 한 줄씩 적는다.
