이름표 없이 묶어도 주기율표가 드러난다 · 팀 짜기

지금까지 다룬 건 모두 동물의 데이터였다. 오늘은 전혀 다른 데이터, 원소 주기율표를 연다. 금속·비금속이라는 이름표를 하나도 주지 않고 성질 숫자만으로 묶어도, 주기율표의 결이 저절로 드러난다. 군집이 동물에만 통하는 재주가 아님을 확인하고, 이어서 팀을 짜고 우리 프로젝트 주제를 정한다.

🎯 오늘 하는 것
  • 원소 군집 데모 — 118개 원소를 이름표 없이 성질만으로 묶어 보기 (선생님 시연)
  • 팀 주제 카드 — 2~3인 팀을 짜고 회귀·분류·군집 중 하나를 골라 주제를 적기

1. 이름표 없이 묶어도 (선생님 시연)

  • 군집은 정답표 없이 비슷한 것끼리 묶는 방법이었다. 동물이 아닌 데이터에서도 통할까?

PubChem이라는 화학 데이터 창고에서 원소 118개의 성질(전기음성도·이온화 에너지·원자 반지름·녹는점)을 가져온다. 여기엔 "이 원소는 금속" 같은 이름표를 넣지 않는다. 성질 숫자만 컴퓨터에 주고 두 묶음으로 나눠 본다.

💡 왜 선생님 시연인가
  • 이번 차시는 "군집이 어디서든 통한다"는 감을 잡는 게 목표. 코드는 선생님 화면으로 함께 본다.
  • 아래 ①②는 눈으로 따라오면 충분하다. 배포는 하지 않는다.
① 샘플 프롬프트
원소 주기율표 데이터를 불러와서, 금속·비금속 같은 분류 이름표는
빼고 성질 숫자만으로 원소들을 두 묶음으로 나눠 줘.

- 데이터 주소: https://pubchem.ncbi.nlm.nih.gov/rest/pug/periodictable/CSV
- 성질은 전기음성도, 이온화 에너지, 원자 반지름, 녹는점을 쓰고
  값이 빠진 원소는 빼 줘.
- 두 묶음에 각각 어떤 원소들이 들어갔는지 기호로 보여 줘.
② AI가 준 코드 예시
import pandas as pd
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans

URL = "https://pubchem.ncbi.nlm.nih.gov/rest/pug/periodictable/CSV"
df = pd.read_csv(URL)

# 성질 숫자만 쓴다 — "금속/비금속" 이름표(GroupBlock)는 넣지 않는다
feats = ["Electronegativity", "IonizationEnergy", "AtomicRadius", "MeltingPoint"]
for c in feats:
    df[c] = pd.to_numeric(df[c], errors="coerce")
a = df.dropna(subset=feats)          # 값이 빠진 원소는 뺀다

Xs = StandardScaler().fit_transform(a[feats])
a = a.assign(묶음=KMeans(n_clusters=2, random_state=42, n_init=10).fit_predict(Xs))

for cl in sorted(a["묶음"].unique()):
    sub = a[a["묶음"] == cl]
    print(f"묶음 {cl} · {len(sub)}개 · 평균 전기음성도 {sub['Electronegativity'].mean():.2f}")
    print("   ", list(sub["Symbol"]))
✔️ 시연에서 확인할 것
  • 값이 온전한 91개 원소가 [66개, 25개] 두 묶음으로 나뉜다.
  • 작은 묶음(25개)엔 H·C·N·O·F·S·Cl 같은 비금속·할로겐·준금속이 대부분 모인다. 전기음성도가 높은 쪽이다.
  • 아무도 "금속/비금속"이라고 알려 주지 않았는데, 성질 숫자만으로 주기율표 오른쪽 위(비금속)가 한 묶음으로 드러났다.
⚠️ 딱 떨어지진 않는다

비금속 묶음에 구리·금 같은 금속 몇 개가 섞이기도 한다. 군집은 완벽한 분류표가 아니라 비슷한 것끼리의 근사한 묶음이다. 그래도 "성질만 보고도 주기율표의 큰 결이 나온다"는 사실은 또렷하다.

💼 데이터로 일하는 사람들 (3분)
  • 데이터 분석가·생명 연구자·기획자·마케터 — 오늘 배운 판단(질문·감식·해석·한계)이 그대로 일이 되는 직업들
  • 관심 진로 분야에도 데이터가 있어요 · 팀 주제를 고를 때 내 진로와 잇는 것도 좋은 선택

2. 팀 짜고 주제 정하기

공통 15차시가 끝나면 팀 프로젝트가 시작된다. 오늘 팀을 짜고 주제의 뼈대를 잡는다. 앞서 배운 세 트랙 중 우리 팀 질문에 맞는 틀을 하나 고른다.

  1. 팀 짜기
    2~3인으로 팀을 만든다. 서로 관심 주제가 통하는 사람과 묶이면 좋다.
  2. 틀 고르기
    우리 질문이 숫자를 맞히는 문제면 회귀, 단계·종류를 맞히는 문제면 분류, 정답 없이 비슷한 것끼리 묶는 문제면 군집이다.
  3. 데이터 후보 적기
    답을 데이터로 확인할 수 있는 질문인지 따져 본다. 쓸 만한 데이터 후보를 3개 적는다.
  4. 주제 카드 제출
    아래 항목을 채워 클래스룸에 제출한다.
💡 우리 팀 주제 카드 (이 항목을 채워요)
  • 팀 이름 · 팀원 (2~3인)
  • 우리 질문 한 문장 — 데이터로 답할 수 있고 범위가 명확하게
  • 고른 틀 — 회귀 / 분류 / 군집 중 하나 + 고른 이유
  • 데이터 후보 3개 — 어디서 구할지도 함께
  • 예상되는 구멍 — 이 데이터에 있을 법한 이상치·결측·편향 하나
⚠️ 답할 수 없는 질문은 피하기

"어떤 동물이 가장 귀여운가" 같은 질문은 취향이라 데이터로 답하기 어렵다. "몸집이 큰 동물은 더 오래 사는가"처럼 측정할 수 있고 범위가 분명한 질문으로 바꾼다.

🎒 오늘의 체크

  • 이름표 없이 성질만으로도 원소가 금속·비금속으로 갈린다는 걸 봤다.
  • 군집이 동물뿐 아니라 다른 데이터에서도 통한다는 걸 안다.
  • 팀(2~3인)을 짰다.
  • 회귀·분류·군집 중 우리 틀을 골랐고 이유를 말할 수 있다.
  • 데이터 후보 3개를 적어 주제 카드를 제출했다.
