전처리 · 왜 로그를 씌우는가

🎯 오늘 만드는 것
  • 로그 눈금 산점도 — 안 보이던 관계가 드러나게 눈금을 바꾼 그래프
  • 전처리 판단 기록 — 무엇을 왜 뺐는지, 왜 로그를 씌웠는지 근거와 함께

0. 지난 차시

  • 몸무게-수명 산점도 · 점이 좌하단에 뭉쳐 관계가 안 보임(r = 0.29, R² = 0.08)
  • 오늘 → 두 가지를 손본다 · ① 빈 값 정리 · ② 눈금을 로그로

1. 빈 값에도 이유가 있다 — 결측 정리

🕵️ 어떤 칸은 왜 비어 있나
  • 몸무게는 4,513종 중 3,641종만, 최대수명은 4,025종만 값이 있다
  • 대사율(metabolic_rate)은 627종만 있다 · 대사율은 재기가 까다롭고 비용이 커서 대부분 비어 있다
  • 즉 결측은 아무렇게나 생긴 게 아니라 측정의 어려움이 남긴 자국 · "없는 값에도 이유가 있다"

오늘 우리 질문은 몸무게와 수명 사이의 관계입니다. 그러니 두 값이 모두 있는 종만 남기면 돼요. 대사율은 이번 분석에 쓰지 않으니 비어 있어도 상관없습니다.

2. 왜 로그를 씌우는가

몸무게는 0.5g(작은 도롱뇽)부터 1억 3,600만 g(대왕고래)까지, 무려 여덟 자릿수를 오르내립니다. 원래 눈금에서는 고래 한 마리가 가로축을 통째로 차지해, 쥐부터 코끼리까지 나머지가 전부 한구석에 겹쳐 버려요.

이때 쓰는 것이 로그 변환입니다. 로그 눈금은 "몇 배 차이"를 같은 간격으로 폅니다. 그래서 20g 쥐와 6만 g 사람의 차이도, 6만 g 사람과 1억 g 고래의 차이도 한 화면에서 나란히 볼 수 있게 돼요. 값을 조작하는 게 아니라, 자릿수를 고르게 펴서 보이게 만드는 것입니다.

3. 정리하고 로그로 다시 그리기 (①②③)

① 샘플 프롬프트
몸무게와 최대수명이 둘 다 있는 종만 남기고, 두 축을 로그 눈금으로 바꿔서
산점도를 다시 그려 줘. 점들이 고르게 퍼져서 관계가 보이게 해 줘.
데이터는 여기야: https://raw.githubusercontent.com/greatsong/modudata/main/data/life_animals.csv
  • 여러분 화면이 이런 모습이면 성공 · 글자까지 같을 필요 없음
② AI가 준 코드 예시
import streamlit as st
import pandas as pd
import numpy as np
import plotly.express as px

st.title("🔬 로그 눈금으로 다시 본 몸무게 vs 수명")

url = "https://raw.githubusercontent.com/greatsong/modudata/main/data/life_animals.csv"
df = pd.read_csv(url)

# 몸무게·최대수명이 둘 다 있는 종만 남기기
a = df.dropna(subset=["adult_weight_g", "max_longevity_yrs"]).copy()
st.write(f"정리 전 {len(df)}종 → 두 값이 다 있는 {len(a)}종")

# 로그(자릿수) 값 만들기
a["logW"] = np.log10(a["adult_weight_g"])
a["logL"] = np.log10(a["max_longevity_yrs"])

# 로그 눈금 산점도
fig = px.scatter(
    a, x="logW", y="logL", hover_name="common_name",
    labels={"logW": "log10(몸무게)", "logL": "log10(최대수명)"},
)
st.plotly_chart(fig, use_container_width=True)
  • 새 그래프가 뜨면 → 세 박자대로 ③ 개선으로 전후를 비교
③ 프롬프트 개선 예시
좋아! 로그로 바꾸기 전 그래프와 바꾼 뒤 그래프를 나란히 놓아서
점들이 어떻게 퍼지는지 한눈에 보이게 해 줘.
✔️ 발견 (★) — 눈금을 바꾸니 관계가 드러난다
  • 정리 전 4,513종 → 두 값이 다 있는 3,188종이 남는다
  • 좌하단에 뭉쳐 있던 점들이 대각선을 따라 고르게 늘어선다 · 무거울수록 오래 사는 흐름이 눈에 들어온다
  • 데이터를 바꾼 게 아니라 눈금만 바꿨는데 안 보이던 관계가 보인다 · 이 직선을 긋는 일은 다음 차시
💡 로그는 왜곡이 아니다
  • 학생 질문 "값을 로그로 바꾸면 데이터를 속이는 거 아닌가요?"
  • 핵심 = 로그는 같은 값을 다른 눈금으로 보는 것 · 순서도 크기 관계도 그대로 · 자릿수만 고르게 편다
  • 몸무게처럼 몇 배씩 커지는 값은 로그로 봐야 관계가 정직하게 드러난다
📝 수행평가 예고 — 전처리 판단 서술
  • 대사율이 대부분 비어 있는 까닭과, 두 값이 있는 종만 남긴 근거를 적기
  • 몸무게에 왜 로그를 씌우는지, 그것이 왜 왜곡이 아닌지 내 말로 설명하기
